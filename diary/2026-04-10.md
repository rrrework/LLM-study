# 1. 大模型是怎么训练出来的？
大模型训练通常分成几个阶段。
第一阶段是预训练。先收集大规模文本数据，比如网页、书籍、代码、问答语料等，经过清洗、去重、过滤后，用自监督学习的方式训练模型。对于大多数生成式大模型来说，核心任务是“给定前文，预测下一个 token”，也就是 next token prediction。模型在这个阶段学到语言规律、常识知识、推理模式和一定的世界知识。
第二阶段通常是指令微调，也叫 SFT。这里会用人工构造或者高质量标注的问答数据，让模型从“会续写文本”变成“会按指令完成任务”。
第三阶段常见的是对齐训练，比如基于人类偏好的 RLHF，或者 DPO 这类 preference optimization 方法。目的是让模型回答更符合人类预期，比如更有帮助、更安全、更稳定。
如果是行业落地场景，还会做第四阶段，也就是领域适配。比如继续预训练、领域微调、知识增强、RAG、工具调用能力训练等，让模型更适合业务场景。

一句话总结就是：先用海量数据做通用能力预训练，再通过指令微调和偏好对齐，把模型变成可用、可控、可落地的助手。

# 2. Transformer 的架构是什么？
Transformer 是目前大模型最核心的基础架构，它最关键的创新是Self-Attention 机制，用注意力替代了传统 RNN 对序列逐步处理的方式。
它的输入一般先经过三步：
文本经过 tokenizer 变成 token id
token id 映射成 embedding
加上位置编码，让模型知道词序信息

Transformer 的核心模块包括：
Multi-Head Self-Attention：让每个位置都能关注序列中其他位置的信息，并且多头机制可以从不同子空间学习不同关系
Feed Forward Network：对每个位置做非线性变换，增强表达能力
Residual Connection + LayerNorm：提升训练稳定性，避免深层网络难训练
如果是完整的 Transformer，还会分成 Encoder 堆叠 和 Decoder 堆叠。如果是 GPT 这类模型，通常只保留 Decoder 部分；如果是 BERT，通常只保留 Encoder 部分。

Transformer 的优势是：并行计算能力强；长距离依赖建模能力强；更适合大规模训练
# 3. Encoder 和 Decoder 是什么？

Encoder 和 Decoder 本质上是 Transformer 里的两类结构。

Encoder：Encoder 的作用是理解输入。
它通常使用双向注意力，也就是说一个 token 在编码时可以同时看到左右两边的上下文，所以更适合理解类任务，比如分类、匹配、抽取、检索表示学习等。BERT 就是典型的 Encoder-only 模型。

Decoder：Decoder 的作用是生成输出。
它使用因果掩码注意力，也就是当前位置只能看到自己前面的 token，不能偷看后面的内容，所以适合做文本生成。GPT 就是典型的 Decoder-only 模型。

Encoder-Decoder：像 T5、BART 这种模型同时有 Encoder 和 Decoder。
Encoder 先对输入做理解，Decoder 再基于 Encoder 的结果一步一步生成输出。它特别适合翻译、摘要、问答这类输入输出转换任务。

一句话概括：Encoder 更擅长理解；Decoder 更擅长生成；Encoder-Decoder 更适合 seq2seq 任务

# 4. Function Call 是怎么训练的？

Function Call 本质上是让模型学会在合适的时候，不是直接输出自然语言，而是输出结构化的工具调用参数。
训练上通常分几部分：
第一部分：监督微调：先构造带标注的数据。数据里会包含：
用户问题
可用工具列表 / function schema
正确的调用时机
正确的函数名
正确的参数 JSON
模型通过 SFT 学会：什么时候该调用工具、该调用哪个工具、参数怎么按照 schema 正确生成
第二部分：格式约束与解码约束：
在推理时，往往会结合 JSON schema、语法约束、function signature，减少输出非法格式的概率。
第三部分：偏好或强化优化：
如果要提升“是否该调工具”“是否乱调工具”“多轮工具调用是否稳定”，会加入偏好数据或者 reward 机制，让模型更倾向于正确调用、少幻觉调用。

Function Call 不是让模型真的“懂函数执行”，而是让它学会：识别用户意图、匹配工具能力、生成符合约束的结构化参数
执行本身还是由外部系统完成的。

# 5. 微调的方案有哪些？

微调方案按资源成本和训练方式，常见有以下几类。
1）全参数微调 Full Fine-tuning
把模型所有参数都更新。
优点是效果上限高；缺点是显存开销大、训练成本高、部署版本管理复杂。
2）参数高效微调 PEFT
只训练很少一部分参数，主流方法包括：
LoRA
QLoRA
Adapter
Prefix Tuning / Prompt Tuning / P-Tuning
其中 LoRA 最常用，它通过对权重矩阵做低秩分解，只训练新增的小矩阵，成本低，效果也不错。QLoRA 则是在量化基础上做 LoRA，进一步降低显存占用。
3）指令微调 SFT
从训练目标上看，SFT 是最常见的微调方式。通过高质量问答或任务样本，让模型学会按指令完成特定任务。
4）偏好对齐
如：RLHF、DPO、PPO 类方法
它们主要解决“回答风格、帮助性、安全性、偏好一致性”的问题。
5）继续预训练 Continued Pretraining
如果是特定领域，比如医疗、金融、法律、代码，也会先拿领域语料继续做预训练，再做 SFT。这样能增强领域语言建模能力。
面试里可以总结一句：
如果资源够、数据强，可以全参；大多数业务场景更常用 LoRA / QLoRA 这类参数高效微调；如果是对齐问题，会叠加 DPO 或 RLHF。

# 7. 大模型分词器是什么？

大模型分词器，也就是 tokenizer，是把原始文本转成模型可处理的最小离散单元 token 的工具，同时也负责把 token 再还原成文本。
它的作用包括：把文本切成 token；给 token 分配唯一 id；控制词表大小和编码效率；影响多语言、罕见词、代码、数字等内容的表示效果
现在大模型通常不是按“词”分，而是按子词或者字节级别来分。常见算法有：BPE、WordPiece、SentencePiece、byte-level BPE
比如一个英文长单词，可能会被拆成几个子词；中文有时一个字一个 token，也可能多个字合成一个 token，具体看 tokenizer 设计。
Tokenizer 很重要，因为它会直接影响：
上下文长度利用率
训练效率
多语言能力
特殊符号和代码的处理效果

# 8. Embedding 是什么？

Embedding 就是把离散符号映射成稠密向量表示。在大模型里，最常见的是把 token id 映射成一个高维向量，这样模型才能进行数值计算。语义相近的内容，在向量空间里通常也更接近。

Embedding 有几层含义：
1）输入层 embedding：token 进入模型时，会先查 embedding table，把 token id 变成向量。
2）语义表示 embedding：在业务系统里，embedding 也经常指句子、段落、文档的语义向量。这个向量可以用来做：语义检索、相似度匹配、聚类、RAG 向量召回
3）为什么重要：因为 embedding 把“符号”变成了“空间中的向量”，模型才能学习词和词、句子和句子之间的语义关系。
你也可以补一句：RAG 里的向量检索，本质上就是先把文档和查询都编码成 embedding，再在向量空间里做近邻搜索。

1. 趋势流和季节流分别是怎么建模的？两者最终如何融合？
论文2.3节详细说明：先通过创新的自适应指数移动平均（AEMA）方法将原始序列分解为非平稳趋势分量和季节分量（公式9-12），克服了传统SMA/EMA固定系数导致的偏差问题。

趋势流（MLP-TimesBlock趋势流，2.4.2节）：专门处理平滑的长期趋势分量。先经过全连接层 + 平均池化 + BatchNorm 进行平滑降维，再集成 TimesBlock 模块（用 FFT 识别最显著的多个周期 → 将一维序列重塑为二维张量 → 2D 卷积捕捉周期内/周期间模式 → 根据 FFT 幅值加权求和自适应聚合），最后通过剩余 MLP 骨干网络输出趋势预测（公式19-20）。
季节流（Patch-TemporalCNN季节流，2.4.1节）：专门处理非线性高频周期性季节分量。借鉴 Vision Transformer 的 Patch 思想，用滑窗将季节分量切割为多个重叠 Patch（公式13），每个 Patch 经过全连接嵌入 + Mish 激活 + 归一化 → 深度卷积（局部动态）+ 逐点卷积（聚合 Patch 间特征）+ 残差连接 → 时间门控卷积单元（两个并行因果膨胀卷积分支）→ 展平 + MLP 头输出季节预测（公式14-18）。

融合方式（2.4.3节）：两个流的预测结果在特征维度上直接拼接，再通过一个最终的全连接层（公式21）生成最终 PM2.5 预测值。没有用 attention 或 gating，而是简单高效的 concat + FC。
2. 自适应混合图结构是怎么实现的？训练时有没有遇到图结构不稳定或过平滑的问题？
论文2.2.1节详细描述了自适应混合邻接矩阵的构建过程（不是单一固定图，而是融合三种信息）：

静态图（基于地理先验）：计算各监测站点欧氏距离，当距离 < 预设阈值（根据北京/天津站点分布特征设定）时边权重为1，否则为0（公式2）。
全局图（捕捉稳定全局相似性）：为每个站点学习一个可训练嵌入向量，通过对称相似度计算任意两个站点的全局关系矩阵（公式3）。
局部图（捕捉有向单向依赖）：用全局嵌入减去另一组独立局部嵌入，得到非对称局部关系矩阵（公式4），增强解释性。

最终混合邻接矩阵 = 静态矩阵 ⊙（全局关系 + 局部关系）（公式5，⊙为元素级乘法）。
得到混合邻接矩阵后，2.2.2节用多头图注意力网络（GAT）动态学习每个邻居的注意力权重（公式6-8），而不是固定权重。
论文中未提及训练时遇到图结构不稳定或过平滑（oversmoothing）的问题，也没有相关实验或解决方案描述。
